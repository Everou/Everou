使用静态工厂方法代替类的构造器

使用静态工厂方法的好处：

1. 有直观的方法名，有方法名更为直观的反馈该方法的作用。
2. 可以复用对象，减少开支。比如单例模式。
3. 可以返回该类的任意子类。这是服务提供者框架的基础

静态工厂方法的缺点：

1. 当类没有非公有的或者受保护的构造方法时,静态工厂无法返回其子类（不明白，如果使用静态内部类提供工厂方法可以解决？）
2. 静态工厂方法并没有过多约定俗成的名字,它与其他的静态方法并没有什么差异

双重检验锁

然而双重检验锁并不是完美的，因为singleton = new Singleton()这个操作并不是原子性的，它所做的又以下几件事情：

1. 堆内存中开辟空间
2. 调用Singleton类的构造方法进行初始化
3. 将内存地址的hashicode赋予引用，此操作保证singleton != null

jvm在不存在happen-before请关系情况下会进行指令重排，无法保障2,3两步的先后顺序。当线程A进入同步块中，并且进行了1,3两步操作,线程B调用了方法，此时singleton != null,直接return singleton,而此时的singleton并没有完成初始化,会抛出异常。解决方法,在1.5之后使用volatile修饰singleton,volatile的作用为:

1. 刷新内存屏障，保证可见性(实际上就是在read一定会在write的操作上)
2. 禁止指令重排
